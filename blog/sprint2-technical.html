Best practices associated with using classes vs. IDs

Before diving into the best practices between using classes and IDs, it is important to understand the difference between the two. A lot of us know about basic tags such as <h1> and <p>, but the basic set of tags arent always going to cover everything. This is where IDs and Classes come in. In this instance we are talking about how they interact with CSS, but these are also used by other languages like Javascript.
The big thing to remember about these two are that ID's are unique, while classes are not. Each element can only have a single ID, and each page can only have one element with that particular ID. Classes can be used on multiple elements, and multiple classes can be applied to the same element.
Its worth noting that elements can have both an ID AND Classes assigned to it. Now onto some best practices.

Stay Classy in CSS

ID's are for uniquely identifying elements. They should be only used when specifically required. IDs also have a much higher specificity than Classes, which can sometimes cause issues within your HTML if you start trying to overwrite IDs with Classes.



Keep it simple, keep it functional!

Overdoing it with IDs or Classes is a great way to make things more complex than they need to be. Trying to find the one line thats misbehaving is less than ideal at any point in a project. Naming: The long and short of it

When it comes to naming your IDs or Classes, make sure its going to be something that doesnt stump you in the future. Trying to remember what .greenthing or #other is in 6 months time isn't going to be fun, and if anyone else is reviewing the code or project, they probably wont be having a great time deciphering it!
